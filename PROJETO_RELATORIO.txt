Nome do cliente: Dr. Fillipe David
Tipo: Landing page profissional para nutricionista + painel administrativo completo
Stack: React 18 + TypeScript + Vite + Tailwind CSS + Supabase
Deploy: Vercel (auto-deploy via push para main no GitHub)
Repositório: https://github.com/gustavodev8/nutri_ladingpage
Domínio: fillipedavidnutri.online (Hostinger, DNS apontando para Vercel)
Vercel URL: https://nutri-ladingpage.vercel.app (ou similar)

O projeto é uma SPA (Single Page Application) com múltiplas rotas. Tem uma área
pública (landing page + páginas de consultas e resultados) e uma área privada
(painel admin em /admin, protegido por autenticação Supabase).

Como funciona o sync de conteúdo:
  1. Na inicialização, carrega do localStorage (cache) imediatamente
  2. Busca no Supabase via fetchContent()
  3. Se Supabase for mais recente (_ts timestamp), sobrescreve
  4. Admin edita → updateContent() → salva no state, localStorage E Supabase
  5. deepMerge() garante que campos novos do DEFAULT sempre existem

resetContent():
  Força o DEFAULT_CONTENT para o Supabase. Botão "Aplicar dados do Dr. Fillipe"
  no AdminDashboard chama essa função. IMPORTANTE: chamar esse botão após deploy
  se o conteúdo do Supabase estiver vazio/desatualizado.

whatsappUrl(message):
  Gera link https://wa.me/{whatsappNumber}?text={message}
  Disponível via useContent().whatsappUrl()

Comportamento inteligente (handleNavClick):
  - type === "page" → navigate(href) via useNavigate
  - type === "anchor" + estamos na home "/" → scrollToAnchor(id) suavemente
  - type === "anchor" + estamos em outra página → navigate("/") + setTimeout(300ms) + scroll
  - Logo clicado: scroll para topo se na home, navigate("/") se em outra página
  - "Agendar Consulta" button → navigate("/consultas")
  - Menu mobile fecha ao trocar de rota (useEffect em location.pathname)

scrollToAnchor(id):
  Calcula posição do elemento, subtrai 80px (altura da navbar) e faz smooth scroll.

  id                  serial PK

  booking_group_id    uuid (agrupa sessões do mesmo paciente/plano)

  total_sessions      int
  client_name         text
  client_email        text
  client_phone        text
  plan_name           text
  plan_index          int
  appointment_date    date (ISO "YYYY-MM-DD")
  appointment_time    text ("09:00")
  type                text ('online' | 'presencial')
  status              text ('pending' | 'confirmed' | 'completed' | 'no_show' | 'cancelled')
  notes               text (opcional)
  completed_at        timestamp (preenchido ao concluir)

availability_slots
  id          serial PK
  date        date
  start_time  text ("09:00")
  type        text ('online' | 'presencial')
  active      boolean

consultation_records
  id                  serial PK
  booking_id          int (FK bookings.id)
  booking_group_id    uuid
  client_name         text
  client_email        text
  notes               text
  weight              numeric
  height              numeric
  next_return_date    date
  next_steps          text
  created_at          timestamp

payment_logs

Funções em src/lib/supabase.ts:
  fetchContent()                    → lê site_content row 1
  saveContent(content)              → upsert site_content row 1
  uploadImage(file)                 → upload para bucket "images/products/"
  uploadPdf(file)                   → upload para bucket "images/pdfs/"
  fetchAvailabilitySlots()          → slots ativos de hoje em diante
  fetchSlotsByMonth(year, month)    → slots de um mês (para calendário admin)
  fetchSlotsByDate(date, type)      → slots de data e tipo específicos
  addAvailabilitySlot(slot)         → inserir slot
  deleteAvailabilitySlot(id)        → deletar slot
  fetchBookings()                   → todos os bookings ordenados por data
  fetchBookingsForDate(date, type)  → horários ocupados (para evitar conflito)
  insertBooking(booking)            → criar agendamento
  updateBookingStatus(id, status, extra?)  → atualizar status + campos extras
  autoCompleteBookings(bookings)    → marca como "completed" bookings confirmed passados
  insertConsultationRecord(record)  → salvar prontuário
  fetchConsultationRecords(groupId) → buscar prontuários por grupo
  confirmBookingsByGroupId(groupId) → confirmar todas as sessões de um grupo

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Arquivo: src/pages/admin/AdminAgendamentos.tsx

Status de agendamento:
  pending     → aguardando confirmação
  confirmed   → confirmado pelo admin
  completed   → consulta realizada (preenchido com prontuário)
  no_show     → paciente não compareceu
  cancelled   → cancelado

Auto-complete na abertura da página:
  autoCompleteBookings() marca como "completed" todos os bookings com status
  "confirmed" cuja data de consulta já passou.

5 abas de filtro:
  Confirmados | Pendentes | Concluídos | Não compareceu | Cancelados

Funcionalidades:
  - Confirmar agendamento (pending → confirmed)
  - Cancelar agendamento (qualquer → cancelled)
  - Marcar como não compareceu (no_show)
  - Reagendar (modal com nova data e hora, chama Edge Function reschedule-booking)
  - Concluir consulta (modal com campos clínicos)

Modal de conclusão (Completion Modal):
  Campos:
    - Peso (kg)
    - Altura (cm) → calcula IMC automaticamente
    - Anotações da consulta (textarea)
    - Próximos passos / encaminhamentos
    - Data do próximo retorno (date picker)
  Ao salvar: updateBookingStatus(id, 'completed') + insertConsultationRecord()

Prontuário (Consultation Records):
  Ao clicar em um agendamento concluído ou de paciente com histórico,
  exibe tabela com todas as consultas anteriores do mesmo booking_group_id.
  Inclui: data, peso, altura, IMC, anotações.
